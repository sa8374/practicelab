# Концепт: Чеклист самопроверки

Где живет:
- Шаг

Что это:
- Структурированный список критериев для самостоятельной проверки работы учеником
- Состоит из: заголовка, описания, упорядоченных элементов (пунктов) с текстом и возможностью отметки
- Инструмент формирования метакогнитивных навыков самооценки и рефлексии

Синонимы:
- Checklist
- Self-assessment checklist
- Проверочный список

Не путать с:
- Инструкция к заданию
  - Почему можно спутать: оба содержат указания для выполнения работы
  - Почему различаем: инструкция говорит что делать, чеклист - как проверить результат
- Критерии оценки
  - Почему можно спутать: оба содержат критерии для оценки работы
  - Почему различаем: критерии оценки используются учителем, чеклист - учеником для самопроверки
- Шаг урока
  - Почему можно спутать: шаг может содержать чеклист как часть задания
  - Почему различаем: шаг - учебная единица с инструкцией, чеклист - инструмент проверки внутри шага
- Работа ученика
  - Почему можно спутать: чеклист используется для проверки работы и часто прикладывается к работе
  - Почему различаем: чеклист - инструмент проверки, работа - объект проверки

Вложенные концепты:
- Элемент чеклиста

Категории:
- По обязательности пунктов:
  - Обязательный чеклист - все пункты должны быть выполнены
  - Рекомендательный чеклист - пункты являются рекомендациями, можно выполнить часть
- По типу проверки:
  - Формальный чеклист - проверка соответствия формальным требованиям (синтаксис, оформление)
  - Содержательный чеклист - проверка качества содержания, логики, аргументации

Примеры:
- Для проверки кода (тип: формальный, обязательность: обязательный):
  1. Код соответствует PEP8/стандартам оформления
  2. Нет синтаксических ошибок
  3. Есть документация к функциям и классам
- Для проверки дизайна (тип: формальный, обязательность: рекомендательный):
  1. Контрастность текста достаточная для чтения
  2. Отступы и выравнивание consistent
  3. Шрифты читаемые на всех устройствах
- Для проверки содержания (тип: содержательный, обязательность: обязательный):
  1. Есть введение, основная часть и заключение
  2. Аргументы подкреплены примерами или цитатами
  3. Нет грамматических и орфографических ошибок
- Для проверки HTML-страницы (тип: формальный, обязательность: обязательный):
  1. Страница валидна по W3C Validator
  2. Все изображения имеют alt-атрибуты
  3. Заголовки иерархически правильно вложены
- Антипример: простое перечисление требований без структуры "проверить/отметить"

Рамки (границы модели ценности):
- Чеклист как инструмент самопроверки, а не автоматической проверки
  - Что за рамкой: автоматические тесты, линтеры, валидаторы
  - Почему ценность сомнительна: это технические инструменты, а не педагогический инструмент формирования навыков самооценки
