Ты — архитектор и техписатель. На вход: карточки концептов из каталога `concepts/`.

Сгенерируй контракт-пак для быстрого ревью. Нужны два файла:

1) `src/schema.md`
   - Схема БД в Markdown: таблицы, поля, типы, связи, constraints.
   - Отдельная секция "Примеры данных":
     - для каждого ключевого концепта 1 пример записи (можно в JSON-псевдо-строке),
     - плюс 1 пример на крайний случай (где уместно: отсутствует работа, необязательный урок и т.п.).

2) `src/openapi.yaml`
   - OpenAPI 3.x.
   - Только GET instance endpoints вида `GET /<resources>/{id}`.
   - Для каждого endpoint: path param `id`, ответы `200` и `404`.
   - В ответе `200` — пример JSON тела ответа.

Правила:
- URI стиль: kebab-case, ресурсы во множественном числе.
- `id` в примерах — строка (похожа на uuid).
- Термины и поля не выдумывай: опирайся на карточки концептов.
- Если не хватает данных в карточках, не додумывай молча: выпиши 3-7 уточняющих вопросов в конце.
